## 1. Node.js の入力処理

```js
const fs = require("fs");
const input = fs.readFileSync(0, "utf-8").trim().split("\n");
```

理解したこと

- `fs` は Node.js でファイルや標準入力を読むための機能
- `readFileSync(0, "utf-8")` で入力全体を文字列として読む
- `.trim()` は前後の余分な空白や改行を削除する
- `.split("\n")` は改行ごとに文字列を分けて配列にする
- `\n` は改行を表す特別な文字

入力例：

```text
5
1 2 3 2 2
2
```

この入力は、内部では次のように扱われる。

```js
["5", "1 2 3 2 2", "2"];
```

## 2. `.split(" ")` の理解

```js
const A = input[1].split(" ").map((x) => Number(x));
```

理解したこと

- `.split(" ")` は半角スペースを目印に文字列を分ける
- 分けた結果は配列になる
- 配列を表示すると、要素の区切りとして `,` が見える
- `,` は自分で作っているのではなく、配列の表示ルールとして見えている

例：

```js
"1 2 3 2 2".split(" ");
```

結果：

```js
["1", "2", "3", "2", "2"];
```

## 3. 特別な文字の理解

`\n` や `\t` のような文字を学習した。

```js
"\n"; // 改行
"\t"; // タブ
" "; // 半角スペース
```

理解したこと

- `\n` は改行
- `\t` はタブ、つまり広めの空白
- `.split("\n")` は改行で分ける
- `.split(" ")` は半角スペースで分ける

## 4. `for...of` と `for` の違い

### `for...of`

```js
for (let value of A) {
  console.log(value);
}
```

これは、配列の中身を1つずつ取り出す。

つまり、値だけ見たいときに使う。

例：

```js
let count = 0;

for (let value of A) {
  if (value === k) {
    count++;
  }
}
```

これは、`k` と同じ値が何個あるか数えるときに使える。

### `for`

```js
for (let i = 0; i < n; i++) {
  console.log(i, A[i]);
}
```

これは、番号 `i` を使って配列を見る。

つまり、何番目かを知りたいときに使う。

例：

```js
let ans = 0;

for (let i = 0; i < n; i++) {
  if (A[i] === k) {
    ans = i + 1;
    break;
  }
}
```

これは、`k` が最初に出てくる位置を調べるときに使える。

## 5. 配列のインデックス

配列は 0 番目から始まる。

```js
const A = [-3, 2, 0, -1, 2];
// i:     0   1   2   3   4
// A[i]: -3   2   0  -1   2
```

でも、問題で聞かれる「何番目」は普通は 1 番目から数える。

そのため、答えを保存するときは次のようにする。

```js
ans = i + 1;
```

## 6. `break` の理解

```js
break;
```

理解したこと

- `break` はループをその場で終了させる
- 無限ループを止めるためだけではない
- 答えが見つかった時点で、それ以上探さないためにも使う

例：

```js
for (let i = 0; i < n; i++) {
  if (A[i] === k) {
    ans = i + 1;
    break;
  }
}
```

`break` がないと、最後に見つかった `k` の位置で答えが上書きされてしまう。

## 7. 問題2：最初に見つかった `k` の位置を求める

問題

サイズ `n` の数列 `A` に含まれる整数 `k` のうち、最も先頭に近いものは数列の何番目にあるか。

コード

```js
const fs = require("fs");
const input = fs.readFileSync(0, "utf-8").trim().split("\n");

const n = Number(input[0]);
const A = input[1].split(" ").map((x) => Number(x));
const k = Number(input[2]);

let ans = 0;

for (let i = 0; i < n; i++) {
  if (A[i] === k) {
    ans = i + 1;
    break;
  }
}

console.log(ans);
```

ポイント

- `for` を使ってインデックス `i` を見る
- `A[i] === k` で目的の値を探す
- 見つけたら `ans = i + 1`
- 最初に見つかった場所だけでよいので `break` する

## 8. 問題3：最初に見つかった偶数の位置を求める

問題

サイズ `n` の数列 `A` に含まれる偶数のうち、最も先頭に近いものは数列の何番目にあるか。

最初に間違えたコード

```js
if (A[i] / 2) {
```

これは偶数判定ではない。

`/` は割り算なので、偶数かどうかの判定には使えない。

例：

```js
4 / 2; // 2
3 / 2; // 1.5
```

どちらも 0 ではないため、`if` の条件を通ってしまう。

正しい偶数判定

```js
A[i] % 2 === 0;
```

`%` は割った余りを求める演算子。

```js
4 % 2; // 0
3 % 2; // 1
```

2で割った余りが 0 なら偶数。

コード

```js
const fs = require("fs");
const input = fs.readFileSync(0, "utf-8").trim().split("\n");

const n = Number(input[0]);
const A = input[1].split(" ").map((x) => Number(x));

let ans = 0;

for (let i = 0; i < n; i++) {
  if (A[i] % 2 === 0) {
    ans = i + 1;
    break;
  }
}

console.log(ans);
```

ポイント

- 偶数判定は `% 2 === 0`
- 最初に見つかった偶数の位置を答える
- 位置を答えるので `for...of` ではなく `for` を使う
- 配列は 0 番目からなので、答えは `i + 1`
- 最初に見つけたら `break` する

## 今日のまとめ

今日は、配列を使った線形探索の基本を学習した。

特に、次の考え方が整理できた。

- `for...of` は値だけを見る
- `for` はインデックスも使って見る

また、配列は 0 番目から始まるため、問題で「何番目」と答えるときは `i + 1` にする必要があると理解した。

さらに、`break` は無限ループを止めるためだけではなく、最初に答えが見つかった時点で探索を終了するためにも使うことを学んだ。

偶数判定では、割り算の `/` ではなく、余りを求める `%` を使う。

```js
A[i] % 2 === 0;
```

つまり、今日の重要ポイントは次の4つ。

- 値を数えるなら `for...of`
- 場所を探すなら `for`
- 最初だけ見つけたいなら `break`
- 偶数判定は `% 2 === 0`

## つまずいた点

- `if (A[i] / 2)` は偶数判定ではないことを途中で整理し直した
- 値を見たいときの `for...of` と、位置を見たいときの `for` の使い分けが最初は少し混ざった
- 配列は 0 番目から始まるのに、答えは 1 番目から数えるので `i + 1` が必要になる点を意識する必要があった
