## 1. `for...of` と `for` の違い

### `for...of`

配列の値そのものを順番に見るときに便利。

```js
for (let value of a) {
  console.log(value);
}
```

この場合、`value` には配列の中身が1つずつ入る。

```js
const a = [1, 2, 3];
// value = 1
// value = 2
// value = 3
```

### `for`

配列の番号（インデックス）を使いたいときに便利。

```js
for (let i = 0; i < n; i++) {
  console.log(a[i]);
}
```

- `i` は配列の番号
- `a[i]` はその番号にある値

```js
const a = [10, 20, 30];
// a[0] = 10
// a[1] = 20
// a[2] = 30
```

つまり、

- 値だけ見たい -> `for...of`
- 番号も使いたい -> `for`

## 2. `n`・`i`・`a[i]` の関係

```js
const n = Number(input[0]);
const a = input[1].split(" ").map((x) => Number(x));
```

`n` は配列の個数。

例えば入力が次のとき：

```text
5
1 2 3 2 2
```

この場合、

```js
n = 5;
a = [1, 2, 3, 2, 2];
```

配列は0番目から始まる。

```js
// a[0] = 1
// a[1] = 2
// a[2] = 3
// a[3] = 2
// a[4] = 2
```

だから、

```js
for (let i = 0; i < n; i++) {
  console.log(a[i]);
}
```

は `a[0]` から `a[4]` までを見る。

`i < n` にする理由は、`n = 5` のとき配列の最後の番号は4だから。

- `i = 0, 1, 2, 3, 4` まで見る
- `i = 5` になったら終了

## 3. 問題1：数列の中に `k` が何個あるか

問題：
整数 `n` と、数列 `a_1, ... , a_n` と、整数 `k` が与えられる。`a_1, ... , a_n` のうち `k` は何個あるかを求める。

コード：

```js
const fs = require("fs");
const input = fs.readFileSync(0, "utf-8").trim().split("\n");

const n = Number(input[0]);
const a = input[1].split(" ").map((x) => Number(x));
const k = Number(input[2]);

let num = 0;

for (let value of a) {
  if (value === k) {
    num++;
  }
}

console.log(num);
```

学んだこと：

`num` はカウント専用の箱。

```js
let num = 0;
```

これは「まだ `k` は0個見つかっていない」という意味。

```js
if (value === k) {
  num++;
}
```

これは「今見ている `value` が `k` と同じなら `num` を1増やす」という意味。

整理すると、

- `value` -> 今見ている数字
- `k` -> 探したい数字
- `num` -> 見つかった個数

## 4. 問題2：`k` が最初に出てくる場所を求める

問題：
サイズ `n` の数列 `A` に含まれる整数 `k` のうち、最も先頭に近いものは数列の何番目にあるか？

コード：

```js
const fs = require("fs");
const input = fs.readFileSync(0, "utf-8").trim().split("\n");

const n = Number(input[0]);
const a = input[1].split(" ").map((x) => Number(x));
const k = Number(input[2]);

let answer = 0;

for (let i = 0; i < n; i++) {
  if (a[i] === k) {
    answer = i + 1;
    break;
  }
}

console.log(answer);
```

学んだこと：

この問題は個数を数える問題ではなく、最初に見つかった場所を答える問題。
だから `for...of` より、番号 `i` を使える `for` が向いている。

```js
if (a[i] === k) {
  answer = i + 1;
  break;
}
```

- 配列の番号は0から始まる
- 問題文の「何番目」は1から数える
- そのため `answer = i + 1` にする
- `break` は最初に見つかったらループを抜けるために使う

## 5. 問題3：最初に出てくる偶数の場所を求める

問題：
サイズ `n` の数列 `A` に含まれる偶数のうち、最も先頭に近いものは数列の何番目にあるか？

コード：

```js
const fs = require("fs");
const input = fs.readFileSync(0, "utf-8").trim().split("\n");

const n = Number(input[0]);
const a = input[1].split(" ").map((x) => Number(x));

let answer = 0;

for (let i = 0; i < n; i++) {
  if (a[i] % 2 === 0) {
    answer = i + 1;
    break;
  }
}

console.log(answer);
```

学んだこと：

偶数かどうかは `%` を使う。

```js
a[i] % 2 === 0;
```

これは「`a[i]` を2で割った余りが0、つまり偶数」という意味。

最初に偶数を見つけたら、

```js
answer = i + 1;
break;
```

とする。

今回も「何番目か」を答えるので、`i + 1` が必要。

## 6. 最大値・最小値の考え方

最大値を探すときは、最初はかなり小さい値を入れておく。

```js
let maxNum = -1000000001;
```

そして、今見ている値が `maxNum` より大きければ更新する。

```js
if (A[i] > maxNum) {
  maxNum = A[i];
}
```

最小値を探すときは逆で、最初はかなり大きい値を入れておく。

```js
let minNum = 1000000001;
```

そして、今見ている値が `minNum` より小さければ更新する。

```js
if (A[i] < minNum) {
  minNum = A[i];
}
```

整理すると、

- 最大値を探す -> 最初はかなり小さい値から始める
- 最小値を探す -> 最初はかなり大きい値から始める

## 7. 今日の重要ポイント

- `for...of` -> 値だけ見たいときに便利
- `for` -> 番号 `i` を使いたいときに便利
- `n` -> 配列の個数
- `i` -> 配列の番号
- `a[i]` -> `i` 番目にある値
- `num` -> 個数を数える箱
- `answer` -> 答えを保存する箱
- `k` -> 探したい数字
- `i + 1` -> 配列の番号を「何番目」に直す
- `break` -> 最初に見つかったらループを抜ける

## まとめ

今日は、線形探索の基本として、

- 配列を先頭から順番に見る
- 条件に合うか調べる
- 見つけたら数える、または場所を保存する

という流れを学んだ。

特に大事なのは、

- 値だけ見るなら `for...of`
- 場所番号も必要なら `for`

問題1は「個数を数える問題」なので `num++`。
問題2・問題3は「最初に見つかった場所を答える問題」なので `answer = i + 1` と `break` を使う。
