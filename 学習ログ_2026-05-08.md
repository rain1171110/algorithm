# 2026-05-08 学習ログ（線形探索で k 番目に大きい値を求める）

## 今日やったこと

今日は、数列 A の中から k 番目に大きい値を求めるアルゴリズムを学習した。

入力例：

5
-9 10 6 0 -3
4

この場合、数列を大きい順に並べると次の通り。

10 6 0 -3 -9

したがって、4 番目に大きい値は -3 になる。

最初は小さい順で考えてしまい、4 番目は 6 だと思っていた。
ただし「k 番目に大きい値」は、最大値から数えて k 番目という意味だと理解した。

---

## 今回の考え方

今回の問題では、配列を実際にソートせず、線形探索を使って求める。

手順：

- 最初に、入力される最大値より大きい値を maxNum に入れる
- maxNum より小さい中で、一番大きい値を探す
- 見つけた値で maxNum を更新する
- これを k 回繰り返す
- 最後に残った maxNum が、k 番目に大きい値になる

## 完成コード

const fs = require("fs");
const input = fs.readFileSync(0, "utf-8").trim().split("\n");

const n = Number(input[0]);
const a = input[1].split(" ").map((x) => Number(x));
const k = Number(input[2]);

let maxNum = 1000000001;

for (let count = 0; count < k; count++) {
let nextMaxNum = -1000000000;

for (let value of a) {
if (value < maxNum && value > nextMaxNum) {
nextMaxNum = value;
}
}

maxNum = nextMaxNum;
}

console.log(maxNum);

---

## コードの意味

### let maxNum = 1000000001;

最初の上限を作っている。
問題文では A[i] <= 1000000000 なので、それより大きい 1000000001 を入れておく。

最初は、

1000000001 未満で一番大きい値を探す

という意味になる。

### for (let count = 0; count < k; count++)

探索を k 回繰り返すループ。

- 1 回目：最大値を探す
- 2 回目：2 番目に大きい値を探す
- 3 回目：3 番目に大きい値を探す
- k 回目：k 番目に大きい値を探す

### let nextMaxNum = -1000000000;

今回見つける最大値候補。
毎回ループの中で「maxNum 未満の中で一番大きい値」を探すために使う。

### for (let value of a)

配列 a の中身を 1 つずつ見る。
ここでの value は、今見ている 1 個の数字。

### if (value < maxNum && value > nextMaxNum)

この条件が重要。

- value < maxNum
  - 前回見つけた値より小さいものだけを見る
- value > nextMaxNum
  - その中で一番大きい値を探す

つまり、

- maxNum より小さい
- かつ
- 今まで見つけた候補 nextMaxNum より大きい

場合だけ nextMaxNum を更新する。

### maxNum = nextMaxNum;

1 回分の探索が終わったら、今回見つけた値を maxNum に入れる。
次の探索では「今見つけた値より小さい中で一番大きい値」を探せるようになる。

---

## 例で確認

入力：

5
-9 10 6 0 -3
4

最初：

maxNum = 1000000001

1 回目：1000000001 未満の最大値を探す -> 10

maxNum = 10

2 回目：10 未満の最大値を探す -> 6

maxNum = 6

3 回目：6 未満の最大値を探す -> 0

maxNum = 0

4 回目：0 未満の最大値を探す -> -3

maxNum = -3

最後に出力される値は -3。

---

## A[i] ≠ A[j] の意味

問題文にある A[i] ≠ A[j] は、配列 A の中に同じ値がないという意味。

A = [-9, 10, 6, 0, -3] の場合：

- A[0] = -9
- A[1] = 10
- A[2] = 6
- A[3] = 0
- A[4] = -3

どの場所を見ても値が重複していない。

今回のコードは value < maxNum で前回と同じ値を除外するため、重複がない前提なら正しく動く。

---

## 重複がある場合の考え方

もし [10, 10, 6, 0, -3] のように同じ数字がある場合、順位の考え方が 2 種類ある。

1. 重複を含める場合

10, 10, 6, 0, -3

- 1 番目: 10
- 2 番目: 10
- 3 番目: 6

2. 重複を無視する場合

10, 6, 0, -3

- 1 番目: 10
- 2 番目: 6
- 3 番目: 0

今回の問題は A[i] ≠ A[j] があるので、重複は考えなくてよい。

### 重複を含める場合の別解

重複も 1 個ずつ数える場合は、値ではなく「場所」を管理する。

const used = Array(n).fill(false);

これは、n 個分の配列を作って、全部 false にするという意味。

例：

const used = Array(5).fill(false);

結果：

[false, false, false, false, false]

その後、

used[1] = true;

とすると、

[false, true, false, false, false]

になる。これは 2 番目の場所だけ使用済みにした、という意味。

## 今日の重要ポイント

- 「k 番目に大きい値」は、最大値から数えて k 番目の値
- 小さい順ではなく、大きい順に考える
- 線形探索では、配列を 1 つずつ見て条件に合えば更新する
- maxNum は「今の上限」
- nextMaxNum は「今回見つける最大値候補」
- value < maxNum && value > nextMaxNum で、次に大きい値を探せる
- A[i] ≠ A[j] は、配列の中に同じ値がないという意味
- 重複がある場合は、値ではなく場所を管理する必要がある
- Array(n).fill(false) は、使用済み管理用の配列を作るときによく使う
