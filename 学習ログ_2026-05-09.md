## 2026-05-09 学習ログ

### 今日やったこと

今日は、JavaScriptで数列の最大値・最小値を求める問題を復習した。

問題内容は、サイズ `n` の数列 `A` の中から、最大値と最小値をそれぞれ求めるもの。

線形探索を使い、配列の要素を先頭から1つずつ確認しながら、暫定の最大値・最小値を更新していく流れを確認した。

---

### 復習したコード

```js
const fs = require("fs");
const input = fs.readFileSync(0, "utf-8").trim().split("\n");

const n = Number(input[0]);
const a = input[1].split(" ").map((x) => Number(x));

let maxNum = -1000000001;
let minNum = 1000000001;

for (let value of a) {
  if (value > maxNum) {
    maxNum = value;
  }

  if (value < minNum) {
    minNum = value;
  }
}

console.log(maxNum, minNum);
```

---

### 学んだこと

#### 1. 標準入力の読み取り

```js
const input = fs.readFileSync(0, "utf-8").trim().split("\n");
```

`readFileSync(0, "utf-8")` は、ターミナルから入力された文字を読み取る処理。

間違えて、

```js
fs.readFileSync("0, utf-8");
```

と書くと、`"0, utf-8"` という名前のファイルを探してしまうためエラーになる。

---

#### 2. 更新する変数は `let` を使う

最大値・最小値は、ループの中で何度も更新する。

そのため、`const` ではなく `let` を使う。

```js
let maxNum = -1000000001;
let minNum = 1000000001;
```

`const` は再代入できないため、

```js
maxNum = value;
minNum = value;
```

のようにあとから値を入れ替える処理には向いていない。

---

#### 3. 初期値は入力範囲の外にする

問題の条件は、

```
-1000000000 <= A[i] <= 1000000000
```

なので、最大値の初期値はそれより小さい値にする。

```js
let maxNum = -1000000001;
```

最小値の初期値はそれより大きい値にする。

```js
let minNum = 1000000001;
```

こうすることで、最初の要素を見た時に必ず最大値・最小値が更新される。

---

#### 4. `for...of` で配列を1つずつ見る

```js
for (let value of a) {
```

これは、配列 `a` の中身を1つずつ `value` に入れて処理する書き方。

例えば、

```js
const a = [3, 8, -2, 10, 4];
```

なら、

```
3 → 8 → -2 → 10 → 4
```

の順番で確認していく。

---

#### 5. 暫定チャンピオンを更新する考え方

最大値は、

```js
if (value > maxNum) {
  maxNum = value;
}
```

今までの最大値より大きい数字を見つけたら更新する。

最小値は、

```js
if (value < minNum) {
  minNum = value;
}
```

今までの最小値より小さい数字を見つけたら更新する。

- `node` は実行する命令
- `a.js` は実行したいファイル

という関係。
