## 2026-05-11 学習ログ

### 今日やったこと

今日は、X円を支払うときに、
500円・100円・50円・10円・5円・1円硬貨を使って、
合計枚数を最小にする問題を復習した。

「大きい硬貨から順番に使う」という方針で、
`for...of` と `Math.floor()`、`%` を使って実装した。

---

### 完成コード

```js
const fs = require("fs");
const input = fs.readFileSync(0, "utf-8").trim().split("\n");

let x = Number(input[0]);

const coins = [500, 100, 50, 10, 5, 1];

let count = 0;

for (const coin of coins) {
  count += Math.floor(x / coin);
  x = x % coin;
}

console.log(count);
```

---

### 学んだこと

#### 1. `coins` 配列

```js
const coins = [500, 100, 50, 10, 5, 1];
```

使える硬貨を配列に入れた。

大きい硬貨から順番に使うことで、硬貨の枚数を少なくできる。

---

#### 2. `for...of` の動き

```js
for (const coin of coins) {
```

`coins` 配列の中身を1つずつ取り出す。

順番は次の通り。

- 500
- 100
- 50
- 10
- 5
- 1

つまり、`coin` には毎回、硬貨の金額が1つずつ入る。

---

#### 3. 一番大事な2行

```js
count += Math.floor(x / coin);
x = x % coin;
```

この2行が今回の中心。

`count += Math.floor(x / coin);` は、

```js
count = count + Math.floor(x / coin);
```

と同じ意味。

`Math.floor(x / coin)` は「今見ている硬貨を何枚使えるか」を求める。

例：`x = 733`、`coin = 500`

```js
Math.floor(733 / 500); // 1
```

つまり、500円玉を1枚使えるということ。

`x = x % coin;` は、その硬貨で払えるだけ払った後の残り金額を更新する処理。

例：

```js
733 % 500; // 233
```

500円玉を1枚使った後の残りは233円。

---

### 733円の場合の流れ

最初

- `x = 733`
- `count = 0`

500円玉

- `733 ÷ 500 = 1` 枚
- `count = 0 + 1 = 1`
- `733 % 500 = 233` → `x = 233`

100円玉

- `233 ÷ 100 = 2` 枚
- `count = 1 + 2 = 3`
- `233 % 100 = 33` → `x = 33`

50円玉

- `33 ÷ 50 = 0` 枚
- `count = 3 + 0 = 3`
- `33 % 50 = 33` → `x = 33`

10円玉

- `33 ÷ 10 = 3` 枚
- `count = 3 + 3 = 6`
- `33 % 10 = 3` → `x = 3`

5円玉

- `3 ÷ 5 = 0` 枚
- `count = 6 + 0 = 6`
- `3 % 5 = 3` → `x = 3`

1円玉

- `3 ÷ 1 = 3` 枚
- `count = 6 + 3 = 9`
- `3 % 1 = 0` → `x = 0`

最終的に `count = 9` なので、出力は `9`。

- `count` は使った硬貨の合計枚数を数える箱

### まとめ

今回の問題は、硬貨を大きい順に見て、
使える枚数を `count` に足し、残り金額を更新していく問題。

特に大事なのは次の2行。

```js
count += Math.floor(x / coin);
x = x % coin;
```

この2行で、
「何枚使えるか」と「次に処理する残り金額」を同時に管理できることを理解した。
